搭个人博客时，我最在意三件事：内容组织是否清晰、部署是否省心、后续维护是否轻量。

Astro 很适合这种场景。文章可以直接用 Markdown 管理，页面默认输出静态文件，部署到 GitHub Pages 也足够顺手。对我来说，这意味着我可以把更多精力放在写内容，而不是被工程细节持续牵着走。

另一个让我喜欢的点，是 Astro 的扩展节奏很舒服。想要简单时，它可以非常简单；想要逐步增加搜索、评论、图表或者 React 组件时，也不会把前面的结构全部推翻。

所以到最后，我还是回到了 Astro。它不一定是最炫的选项，但对“长期写博客”这件事来说，确实足够稳。
