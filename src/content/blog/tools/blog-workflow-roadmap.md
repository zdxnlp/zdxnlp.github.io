我不想把博客维护成一个“每次改点东西都很累”的项目，所以准备先把工作流定得尽量简单一些。

目前的计划是这样：

1. 文章直接写在 `src/content/blog/` 里，用 Git 管理版本。
2. 页面样式优先保持稳定，不频繁大改结构。
3. 每次改完内容后直接推到 GitHub，让 Actions 自动部署。

这样做的好处是，写作和发布之间几乎没有额外步骤。只要打开项目、写完、提交、推送，站点就会自动更新。

对个人博客来说，流程越短，越容易坚持。先让它稳稳跑起来，再考虑更复杂的功能，会比一开始追求“大而全”更适合长期维护。
